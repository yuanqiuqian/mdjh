# 对话型武侠文字游戏（PWA）设计概览

## 1. 产品定位

- 类型：对话型文字游戏（以“对话选择 + 事件反馈 + 长线成长”为核心）
- 题材：武侠江湖
- 平台：Web（优先移动端），支持PWA安装与离线可玩（离线限定为“已有内容回放/本地模拟”，AI剧情需联网）
- 特色：AI实时控制剧情走向，并可生成新角色；玩家可无限续玩、无玩法上限（生存、修行、赚钱、社交/恋爱等）

## 2. 核心目标与成功标准

- 玩家目标：在江湖上活下去（死亡即结束，除非有救助类药物触发保命）
- 系统目标：
  - 让玩家在“自由选择”与“可持续推进”之间保持平衡
  - AI叙事可控：保证连贯、可回溯、可压缩、可复盘
  - 存档可靠：支持多存档与回滚，避免一次异常对话导致不可逆损坏
  - PWA体验合格：可安装、启动快、弱网可用（至少UI与存档可用）

## 3. 核心循环（建议）

1) 触发情境：地点/时间/事件（由AI或系统调度）
2) 玩家输入：对话/选择（自然语言输入 + 可选按钮作为引导）
3) AI生成：剧情推进 + 新事件 + 角色行动 + 数值变化
4) 系统结算：属性/物品/金钱/好感度/状态变化，检查死亡与救助规则
5) 记录与压缩：写入结构化事件；超阈值时压缩旧记录并生成摘要
6) 下一回合：进入新的对话或事件

## 4. 玩法范围与边界（第一阶段）

包含：
- 8大门派起始选择与初始武器差异
- 角色属性（等级、基础属性、三维属性、技能、角色类型与攻击加成）
- 好感度/仇恨系统（可为负）
- 主线：固定开场（10轮对话性格引导）+ 后续AI开放剧情
- 存档/读档
- 上下文窗口管理（结构化记录 + 动态压缩）
- PWA壳：安装、离线启动、基础资源缓存、本地存档

不包含（暂缓）：
- 复杂的地图可视化、实时战斗动画
- 多人联机与交易行
- 深度门派任务链（可先作为AI可生成任务）

## 5. 固定开场剧情（第一幕）

设定：
- 主角：16岁，初始等级1，初出茅庐，刚下山
- 事件：遇到山贼打劫路人
- 选择倾向示例：出手相助 / 视而不见 / 帮助山贼 等
- 目标：通过10轮对话完成“性格引导”，建立主角行为偏好画像

产出：
- 性格画像（结构化标签 + 自然语言摘要）
- 初始人际关系（至少：路人/山贼首领等，含好感度）
- 初始资源变化（可能获得金钱/物品/仇恨）
