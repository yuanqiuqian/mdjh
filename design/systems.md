# 系统设计（属性/门派/战斗/好感度/物品/存档）

## 1. 主角与等级

- 初始：16岁，等级1
- 等级用途：提升基础属性上限与成长；解锁技能/装备门槛（可选）
- 升级资源：经验（EXP），主要来自战斗、任务、修行、事件奖励（由AI与系统共同结算）

## 2. 角色属性模型

### 2.1 三维属性（主属性）

- 力量 STR：影响血量上限
- 敏捷 AGI：影响攻速、护甲
- 智力 INT：影响内力上限

### 2.2 基础属性（战斗属性）

- 血量 HP：0以下触发死亡判定
- 内力 MP：技能消耗资源
- 攻击力 ATK：普攻与部分技能基础
- 攻速 ASPD：决定行动频率/先手（见战斗规则）
- 护甲 ARM：减伤

### 2.3 技能

- 每个角色可拥有多个技能
- 技能构成：名称、类型（伤害/控制/治疗/增益/减益/位移等）、消耗、冷却、作用对象、数值公式、描述

## 3. 角色类型与攻击加成

角色按成长方向分为三种类型：

- 力量型：ATK主要受STR影响
- 敏捷型：ATK主要受AGI影响
- 智力型：ATK主要受INT影响

建议的统一公式（便于平衡与可扩展）：

- HP_max = HP_base + STR * hp_per_str + LV * hp_per_lv
- MP_max = MP_base + INT * mp_per_int + LV * mp_per_lv
- ATK = ATK_base + LV * atk_per_lv + Primary * atk_per_primary
- ASPD = ASPD_base + AGI * aspd_per_agi
- ARM = ARM_base + AGI * arm_per_agi

其中 Primary 为角色类型对应主属性（力量型=STR，敏捷型=AGI，智力型=INT）。

建议初始参数（可后续调优）：

- hp_per_str = 10
- mp_per_int = 8
- atk_per_primary = 2
- aspd_per_agi = 0.02（等价于每50敏捷+1次行动频率的量级）
- arm_per_agi = 0.2

## 4. 八大门派

### 4.1 门派选择

主角初始可从以下八个门派中任选其一：

- 武当、少林、峨眉、昆仑、崆峒、华山、明教、丐帮

每个门派定义：

- 门派类型（力量/敏捷/智力）
- 擅长武器（影响初始武器与技能倾向）
- 门派特性（1条被动或开局优势）
- 初始技能（1个）与可习得技能池（后续扩展）

### 4.2 门派类型（建议版，可调整）

- 武当：智力型（以“内功/剑意/以柔克刚”为核心）
- 少林：力量型（以“外功/拳掌/金刚护体”为核心）
- 峨眉：智力型（偏“心法/剑法/疗愈或控制”）
- 昆仑：敏捷型（偏“身法/快剑/寒劲”）
- 崆峒：智力型（偏“奇门/内劲/点穴”）
- 华山：敏捷型（偏“剑术/爆发/决斗”）
- 明教：力量型（偏“狂暴/重击/群战”）
- 丐帮：敏捷型（偏“棍法/连击/控制与游斗”）

### 4.3 初始武器（建议版）

- 武当：长剑
- 少林：禅杖
- 峨眉：双刺/软剑（二选一）
- 昆仑：长剑
- 崆峒：短剑/判官笔（二选一）
- 华山：长剑
- 明教：大刀
- 丐帮：长棍

## 5. 装备与物品

### 5.1 装备栏

- 头部
- 身体
- 武器
- 饰品
- 鞋子

装备属性建议分层：

- 主属性加成：STR/AGI/INT
- 战斗属性加成：HP/MP/ATK/ASPD/ARM
- 特效：触发类被动（如“受致命伤时消耗一次救助丹药”）

### 5.2 物品栏与金钱

- 物品类型：消耗品、材料、任务物、秘籍、丹药（救助类属于消耗品子类）
- 金钱：用于交易、住宿、请教、修行消耗等（由AI剧情与系统事件结算）

## 6. 战斗与死亡规则（对话/交战双模式）

### 6.1 模式切换

- 默认处于对话模式：玩家通过自然语言与建议选项推进剧情
- 当AI识别到明确冲突升级（如攻击、拔刀、围杀、追击失败等）时，系统切换到交战模式
- 交战模式中不再逐招请求AI，而由本地战斗引擎负责回合制结算
- 战斗结束后，系统将战报摘要回灌给AI，由AI续写战后剧情

### 6.2 交战模式行动项

- 进攻：对选定目标进行普通攻击
- 防御：本回合显著降低受到的伤害
- 技能：消耗MP施放门派招式或调息技能
- 道具：使用消耗品恢复状态或触发特殊效果
- 逃跑：根据局势与攻速进行脱战判定

### 6.3 行动与攻速

建议采用“回合内行动点”简化：

- 每个小回合，双方各累积行动值 AV
- AV增长 = ASPD
- 当 AV >= 1 时可行动一次，行动后 AV -= 1
- 直到一方HP <= 0 或双方脱离战斗条件成立

当前实现简化版：

- 玩家每回合主动选择一次行动
- 敌方在同回合依序自动执行行动
- 攻速当前主要用于决定逃跑成功率与后续扩展的先手/节奏判断

### 6.4 伤害与减伤（建议）

- 伤害 = max(1, ATK * SkillMultiplier - ARM * armor_factor)
- armor_factor 建议为 0.3（后续可按体验调整）

当前实现补充：

- 防御状态下，受到伤害按系数减免
- 技能区分为伤害型与恢复型
- 敌方优先使用基础攻击，满足条件时会施放技能

### 6.5 战斗结束与战利品

- 结束条件：
  - 敌方全灭：胜利
  - 玩家HP归零：落败
  - 逃跑成功：脱离战斗
- 战后结算内容：
  - 玩家当前HP/MP同步回主存档
  - 战斗日志压缩成摘要写入事件记录
  - 战利品与金钱落袋
  - 关系变化落盘
  - 自动触发AI续写战后剧情

战利品策略（当前实现）：

- 首领战可掉落关键任务物（如山寨腰牌）
- 山贼类敌人可掉落基础消耗品（如止血散）
- 同时掉落少量碎银

### 6.6 死亡与救助

- 当 HP < 0 且物品栏中不存在“特殊救助类药物”（如大还丹、九转金丹等）可用，则游戏结束
- 若存在可用救助类药物：
  - 自动或提示玩家是否使用（由交互策略决定）
  - 使用后：HP被拉回至阈值（如 1 或 10% HP_max），并记录一次“死里逃生”事件

## 7. 好感度/仇恨

- 每个NPC对主角维护一个好感度值 Favor（允许为负）
- Favor < 0 时可视为仇恨/敌对倾向强
- Favor变化来源：对话态度、利益交换、战斗、救助、背叛等

建议Favor区间与叙事映射：

- 80 ~ 100：生死之交
- 30 ~ 79：友好
- -29 ~ 29：中立/观望
- -79 ~ -30：敌视
- -100 ~ -80：死仇

战斗后的关系变化原则：

- 击败敌方首领：敌对关系可直接进入“死仇”
- 击退普通敌人：敌视程度上升，但允许后续留出复仇或求饶分支
- 保护路人或盟友成功：对应友方好感上升
- 临阵脱逃或战斗失利：旁观者与护送对象可能降低对主角的信任

## 8. 存档/读档（设计层）

存档目标：

- 支持多存档位
- 支持读档回滚
- 防止AI异常输出导致存档损坏（可保留“上一次稳定点”）

建议存档内容：

- 主角快照：等级、属性、装备、物品、金钱、状态、性格画像摘要
- 当前模式：dialogue / combat
- 若处于交战模式：战斗快照（敌我名单、回合数、日志、目标）
- 世界状态：当前位置、时间、已知阵营关系、关键flag
- 角色关系表：已遇到的NPC清单 + Favor
- 事件日志：最近N条详细记录 + 更早的压缩摘要
