# PWA与信息架构（IA）设计稿

## 1. PWA目标

- 可安装：支持“添加到主屏幕”，具备独立启动图标与启动页
- 启动快：首屏为“游戏壳”（菜单/继续游戏/读档），不依赖联网即可展示
- 弱网可用：网络差时可进入只读回放/管理存档；联网恢复后继续推进剧情
- 数据安全：存档本地优先；可选云同步作为后续能力

## 2. 核心页面与导航结构

建议采用底部导航（移动端）+ 侧滑设置（可选）：

- 首页（Home）
  - 新游戏
  - 继续（默认进入最近存档）
  - 读档管理
  - 设置（声音/字体/网络/隐私）
- 新游戏流程（New Game）
  - 门派选择（8选1）
  - 初始信息确认（年龄16、等级1、初始武器、门派特性）
  - 进入开场剧情（固定事件）
- 游戏主界面（Game）
  - 对话模式：
    - 对话区（叙事文本流）
    - 输入区（文本输入 + 快捷选项按钮）
    - 状态条（HP/MP/金钱/当前地点）
    - 事件提示（战斗/受伤/获得物品）
  - 交战模式：
    - 敌我面板（HP/MP/目标选择）
    - 操作面板（进攻 / 防御 / 技能 / 道具 / 逃跑）
    - 战斗日志（本场回合回放）
- 角色（Character）
  - 等级/三维属性/基础属性/技能列表
  - 角色类型与加成说明
- 装备与物品（Inventory）
  - 装备栏（头/身/武器/饰品/鞋）
  - 物品栏（分类筛选：消耗品/材料/秘籍/任务）
  - 金钱与交易入口（后续）
- 人物关系（Relations）
  - 已遇到NPC列表（按favor排序）
  - NPC详情（favor趋势、关键事件摘要）
- 存档（Saves）
  - 多存档位
  - 导入/导出（导出为文件以便迁移）
  - 回滚到上一次稳定点
- 日志与回放（Log）
  - 最近事件列表（可搜索）
  - 压缩摘要视图（章节化）

## 3. 关键交互：对话与选择

输入策略（建议混合）：

- 默认：玩家自然语言输入
- 同时提供AI建议选项（按钮），用于降低输入门槛与保证推进
- 长对话时提供“总结当前局势”按钮（触发AI输出摘要但不推进剧情，或仅推进极小）

对话轮的UI规范：

- AI叙事：分段呈现；重要结果（受伤/获得物品/关系变化）以强调样式提示
- 玩家输入：气泡与AI区分；可在日志中回看
- 风险提示：对高风险选项可标注“可能战斗/可能失去金钱”等（由AI给出risk字段）

交战模式的UI规范：

- 明确展示当前处于“交战模式”，避免玩家误以为仍是普通对话
- 始终展示战斗目标、当前回合、敌方剩余血量
- 技能与道具应显示核心信息（消耗/效果）
- 战斗结束后，界面可直接进入“战后续写中”的过渡态，不要求玩家手动再点一次推进

## 4. 离线策略（设计层，不涉及实现细节）

离线可用的功能（建议必须支持）：

- 打开应用壳与查看本地存档
- 回放历史日志与摘要
- 查看角色/物品/关系页面
- 本地交战中的继续操作与结算（不依赖联网）

离线不可用的功能（合理限制）：

- 推进新的AI剧情/对话（需要联网调用大模型）
- 战后自动续写与新战斗初始化（需要联网调用大模型）

离线时的体验建议：

- 游戏主界面显示“离线只读模式”
- 输入框禁用或改为“记录想法（不影响剧情）”
- 提供“网络恢复后继续”引导

## 5. 存档与数据持久化（与PWA结合）

本地优先：

- 存档以“快照 + 日志（RecentLog + LongSummary）”结构存储
- 写入应原子化：避免浏览器崩溃导致半写入

导入/导出（建议）：

- 导出为单文件（包含多个存档位）或按存档位导出
- 导入时做校验（版本号、字段缺失兜底）

## 6. 隐私与安全（面向用户的承诺）

- 不收集真实身份信息作为必填
- 对话内容默认仅用于生成剧情，不用于外显分享
- 若后续有云同步：明确告知存储位置、加密策略与删除入口
